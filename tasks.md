# Week-1 Tasks: backtest-platform

This file contains your structured checklists for the first week of implementation. You can edit this file to mark items as completed (`[ ]` -> `[x]`).

## Task 1: Setup Development Environment
- [ ] Create Python virtual environment (`python -m venv .venv`).
- [ ] Install dependencies in development/editable mode (`uv pip install -e .` or `pip install -e .`).
- [ ] Install test dependencies (`pytest`).
- [ ] Run `pytest` to verify the smoke test passes.

## Task 2: Define Event Data Classes
- [ ] Create `src/backtest_platform/events.py`.
- [ ] Implement data classes or Pydantic models for:
  - `MarketEvent`: carries bar data.
  - `SignalEvent`: signal from strategy.
  - `OrderEvent`: order generated by portfolio.
  - `FillEvent`: executed trade details.

## Task 3: Implement Backtest Engine / Event Loop
- [ ] Create `src/backtest_platform/engine.py`.
- [ ] Create an `Engine` class containing:
  - An event queue.
  - An event loop that pops events and dispatches them to their respective handlers.

## Task 4: Implement a Simple Strategy
- [ ] Create `src/backtest_platform/strategy.py`.
- [ ] Implement a simple strategy (e.g., Simple Moving Average crossover) that consumes `MarketEvent` and generates `SignalEvent`.

## Task 5: Implement execution & Portfolio Simulators
- [ ] Create `src/backtest_platform/execution.py` (simulates order matching and fees/slippage).
- [ ] Create `src/backtest_platform/portfolio.py` (tracks holdings, cash, and PnL).

## Task 6: Implement CLI & Output
- [ ] Update `src/backtest_platform/cli.py` to run the backtest end-to-end on a CSV input.
- [ ] Output a list of executed trades and summary metrics (Total Return, Max Drawdown).

## Task 7: Add Flow Tests
- [ ] Write integration/unit tests under `tests/` verifying the event flow behaves deterministically and without look-ahead bias.
